二分查找

1. 简介#

二分查找是一种基于分治策略的高效搜索算法，它利用数据的有序性，每轮缩小一半搜索范围，直至找到目标元素或搜索区间为空为止

1.1. 流程#

  1. 定义两个指针left和right,分别指向数组的开始位置和结束位置
  2. 中间的位置的元素mid=left+(right-left)/2
  3. 判断nums[mid]和target的大小关系
    1. 当 nums[mid] < target 时，说明 target 在区间 [mid+1,right] 中，因此执行left=mid+1
    2. 当 nums[mid] > target 时，说明 target 在区间 [left,mid−1] 中，因此执行right=mid-1
    3. 当 nums[m] = target 时，说明找到 target ，因此返回索引 mid 若数组不包含目标元素，搜索区间最终会缩小为空。此时返回 −1

1.2. 复杂度#

时间复杂度为 O(log⁡n) ：在二分循环中，区间每轮缩小一半，因此循环次数为 log2⁡n  空间复杂度为 O(1) ：指针 left 和 right 使用常数大小空间

2. 题目#

704 二分查找-力扣 给定一个 n 个元素有序的（升序）整型数组 nums 和一个目标值 target  ，写一个函数搜索 nums 中的 target，如果目标值存在返回下标，否则返回 -1

示例 1:

1输入: nums = [-1,0,3,5,9,12], target = 9     
2输出: 4       
3解释: 9 出现在 nums 中并且下标为 4

示例 2:

1输入: nums = [-1,0,3,5,9,12], target = 2     
2输出: -1        
3解释: 2 不存在 nums 中因此返回 -1

提示：

  • 你可以假设 nums 中的所有元素是不重复的
  • n 将在 [1, 10000]之间
  • nums 的每个元素都将在 [-9999, 9999]之间

3. 代码实现#

二分法最重要的两个点：

  • while循环中 left 和 right 的关系，到底是 left <= right 还是 left < right
  • 迭代过程中 mid 和 right 的关系，到底是 right = mid - 1 还是 right = mid

3.1. 第一种写法左闭右闭#

二分查找-左闭右闭动画演示

定义target是在[left,right]区间：

  • while (left <= right) 要使用 <= ，因为left == right是有意义的，所以使用 <=
  • if (nums[mid] > target) right 要赋值为 mid - 1，因为当前这个nums[mid]一定不是target，那么接下来要查找的左区间结束下标位置就是 mid - 1
 1class Solution {  
 2public int search(int[] nums, int target) {  
 3	//初始化双闭区间 [0, n-1] ，即 left, right 分别指向数组首元素、尾元素
 4        int left = 0;
 5        int right = nums.length - 1;
 6
 7
 8        while (left <= right) {  //循环的条件
 9            int mid = left + (right - left) / 2;//查找数组的中间元素
10            if (nums[mid] < target) { //说明target在[mid+1,right]中
11                left = mid + 1;
12            } else if (nums[mid] > target) { //明target在[left,mid-1]中
13                right = mid - 1;
14            } else { //说明找到目标值，返回索引
15                return mid;
16            }
17        }
18        //说明没有找到目标值，返回-1
19        return -1;
20}

3.2. 3.1.1.第二种写法左闭右开#

二分查找-左闭右开动画演示

定义target在[left,right) 区间：

  • while (left < right)，这里使用 < ,因为left == right在区间[left, right)是没有意义的
  • if (nums[mid] > target) right 更新为 mid，因为当前nums[middle]不等于target，去左区间继续寻找，而寻找区间是左闭右开区间，所以right更新为mid，即：下一个查询区间不会去比较nums[mid]
 1class Solution {  
 2public int search(int[] nums, int target) {  
 3  // 初始化左闭右开区间 [0, n) ，即 left, right 分别指向数组首元素、尾元素+1
 4        int left = 0;
 5        int right = nums.length;
 6
 7        while (left < right) {
 8            int mid = left + (right - left) / 2;
 9            if (nums[mid] < target) {//target 在右区间，在[mid, right)中
10                right = mid;
11                left = mid + 1;
12            } else if (nums[mid] > target) {//target 在左区间，在[left, middle)中
13                right = mid;
14            } else {  //说明找到目标值，返回索引
15                return mid;
16            }
17        }
18        //说明没有找到目标值，返回-1
19        return -1;				
20}
